# Как было?

## 1. Проблемы

- БД хранится в виде файла workouts.db, используется SQLite.
- Только один пользователь (авторизация по паролю: любой, кто знает пароль, имеет доступ к одной и той же бд).
- Бот возвращает только историю выбранного упражнения, нет функции вывода всех тренировок, хотя в бд такая таблица есть.
- Таблица (лог упражнения) при большом количестве подходов смещается и выглядит ужасно.
- У пользователя нет возможности добавить новые упражнения, только использовать заранее добавленые в csv-файл.

# 2. Что сделано?

## Архитектурно

- Переход с Java на Python (так как библиотеки для создания ботов в Python считаются лучше и Python сейчас для меня более актуален).
- Переход с SQLite на PostgreSQL.
- Использован Docker Compose (потому что это удобно, и я не знаю, как запускать postgres без докер-контейнера).
- Добавлена модель пользователя и переосмыслены остальные модели.

## Функционально

- Исправлен вывод таблиц (это уже не таблицы).
- Добавлена возможность смотреть все воркаут-сессии.
- Добавлены пометки к воркаут-сессии (например, если используется периодизация нагрузки, то какая сейчас неделя).

# Модели БД

## users

- id BIGINT (telegram user id)

## exercises

- id SERIAL PK
- name VARCHAR(128)
- user_id BIGINT FK → users.id -- кастомные упражнения пользователя (видит только этот пользователь)

## muscles

- id SERIAL PK
- name VARCHAR(64) -- "грудь", "трицепс", "квадрицепс"
- user_id BIGINT FK → users.id -- кастомные мышцы пользователя (видит только этот пользователь)

## exercise_muscles -- many-to-many

- exercise_id INT FK → exercises.id
- muscle_id INT FK → muscles.id

## workouts

- id SERIAL PK
- user_id BIGINT FK → users.id
- date DATE
- notes TEXT nullable

## workout_exercises -- какие упражнения были на тренировке

- id SERIAL PK
- workout_id INT FK → workouts.id
- exercise_id INT FK → exercises.id
- position SMALLINT -- порядок упражнений в тренировке

## sets -- конкретный подход

- id SERIAL PK
- workout_exercise_id INT FK → workout_exercises.id
- reps SMALLINT -- количество повторений
- weight NUMERIC(5,2) nullable -- NULL = bodyweight

# Функции бота

## 0. /start

- Краткое описание всех команд бота.

## 1. /exercises — Упражнения

- Вывести все упражнения.
- Найти упражнение по частичному совпадению названия.
- Отфильтровать по группе мышц.
- Добавить новое упражнение.
- Удалить упражнение, добавленное пользователем.

## 2. /history — История тренировок

- Вывести список тренировок с датами.
- При выборе тренировки — показать упражнения, подходы и заметки (детализация по нажатию / пагинация).

## 3. /workout — Запись тренировки

**Старт:**

- Две inline-кнопки: `Add exercise` и `Finish session`.

**Добавление упражнения:**

1. Пользователь вводит название — при частичном вводе выводятся совпадения.
2. После выбора упражнения выводится лог предыдущих результатов (50 последних подходов и дат этого упражнения с прошлых тренировок)
3. Бот ждёт ввода подходов в формате `100x5` (вес × повторения) или `BWx10` (bodyweight). При неправильном вводе, пользователю напоминают правильный способ ввода.
4. После каждого подхода появляется кнопка `Finish exercise` — можно продолжить ввод или завершить.
5. После завершения упражнения снова появляются кнопки `Add exercise` и `Finish session`.

**Завершение тренировки:**

1. Бот спрашивает дату — сегодня или ввести вручную.
2. Бот спрашивает заметки — добавить или пропустить.
3. Данные сохраняются в БД одной транзакцией (atomic commit).

**Поведение в особых случаях:**

- **Отмена:** можно отменить текущее упражнение или всю тренировку.
- **Защита от дублей:** повторный `/workout` во время активной записи триггерит предупреждение, что тренировка уже идёт.
- **Асинхронность работы:** доступ к `/exercises` и `/sessions` во время тренировки работает штатно. Состояние текущей тренировки удерживается в FSM и продолжается после использования других команд.
