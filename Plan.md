# Что сейчас?

## 1. Проблемы

- БД хранится в виде файла workouts.db, используется SQLite.
- Только один пользователь (авторизация по паролю: любой, кто знает пароль, имеет доступ к одной и той же бд).
- Бот возвращает только историю выбранного упражнения, нет функции вывода всех тренировок, хотя в бд такая таблица есть.
- Таблица (лог упражнения) при большом количестве подходов смещается и выглядит ужасно.
- У пользователя нет возможности добавить новые упражнения, только использовать заранее добавленые в csv-файл.

# 2. Что планируется сделать?

## Архитектурно

- Перейти с Java на Python (так как библиотеки для создания ботов в Python считаются лучше и Python сейчас для меня более актуален).
- Перейти с SQLite на PostgreSQL.
- Использовать Docker Compose (потому что это удобно, и я не знаю, как запускать postgres без докер-контейнера).
- Добавить модель пользователя и подумать над остальными моделями.

## Функционально

- Исправить вывод таблиц.
- Добавить возможность смотреть все воркаут-сессии.
- Добавить пометки к воркаут-сессии (например, если используется периодизация нагрузки, то какая сейчас неделя).

## 1. Инфраструктура и базовый бот

5. Написать docker-compose.yaml

## 2. Проектирование БД и миграции

1. Спроектировать нужные сущности
2. Установить библиотеки для БД (asyncpg, alembic, sqlalchemy)
3. Описать модели в коде
4. Сгенерировать первую миграцию
5. Написать скрипт первичной загрузки базовых упражнений из CSV файла

## 3. Базовая логика и авторизация

1. Создать middleware в aiogram, который при каждом сообщении проверяет, есть ли пользователь в таблице users, и если нет — автоматически его создает (регистрация по telegram_id).
2. Написать класс-репозиторий для работы с БД (методы create_session, add_exercise, add_set).
3. Реализовать команду /start с главным меню

## 4. Реализация бизнес-логики

1. Управление упражнениями: Реализовать диалог (через FSM aiogram) добавления пользователем своего собственного упражнения.
2. Процесс тренировки: Реализовать флоу записи подходов: "Начать тренировку" -> "Выбрать упражнение" -> "Ввести вес и повторения".
3. Заметки: Добавить шаг FSM в начале или конце тренировки для добавления текстовой пометки
4. Форматирование вывода: Изучить HTML-форматирование в Telegram или библиотеку prettytable для Python.
   Переписать логику вывода лога упражнения так, чтобы он не "плыл" на мобильных экранах (например, отказаться от таблиц в пользу форматированных списков).
5. История: Реализовать команду /history, которая выводит список всех завершенных воркаут-сессий пользователя с пагинацией (кнопки "вперед/назад").
